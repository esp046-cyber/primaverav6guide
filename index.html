Calendar Configuration Guide

Understanding P6 Calendars

Calendars define when work can be performed. P6 uses calendars to calculate activity dates, durations, and resource availability. Proper calendar setup is critical for realistic oil & gas scheduling.

Calendar Types

Calendar Type Purpose Example
Global Calendar Enterprise-wide, reusable across all projects Standard 5-day week, Offshore 7-day continuous
Project Calendar Specific to one project, inherits from global Project with unique shutdown periods
Resource Calendar Individual resource availability Specialist on rotation schedule

Creating Global Calendars (Admin)

1Access Calendar Manager:

  • Enterprise ‚Üí Calendars
  • Or Admin ‚Üí Calendars for administrative view

2Create New Calendar:

  • Click Add button
  • Enter Calendar Name (e.g., "Offshore 24/7 Continuous")
  • Select Type: Global or Project

3Define Standard Workweek:

  • Click Workweek tab
  • For each day (Mon-Sun), set work hours:
    • Working day: Check box, enter start/end times (e.g., 08:00-17:00)
    • Non-working day: Uncheck box
  • Define break times if needed (lunch breaks typically excluded)

4Add Exceptions (Holidays/Shutdowns):

  • Click Exceptions tab
  • Click Add button
  • Enter exception date and description
  • Set as non-working or define special hours
  • Repeat for all company holidays, maintenance windows

5Calculate Work Hours Per Time Period:

  • P6 auto-calculates hours per day, week, month, year
  • Review in Calendar details to verify accuracy
  • Critical for duration calculations

Standard Oil & Gas Calendar Configurations

Offshore Continuous Operations

Use Case: Platform construction, subsea installation, marine operations

  • Workweek: Monday-Sunday, all days working
  • Hours: 24 hours per day (00:00-24:00)
  • Work Hours/Day: 24
  • Exceptions: Severe weather downtime windows (if predictable)
  • Notes: Use for activities that run round-the-clock with crew rotations

Onshore 6-Day Operations

Use Case: Pipeline construction, field development, fabrication yards

  • Workweek: Monday-Saturday working, Sunday off
  • Hours: 07:00-17:00 (10-hour days typical)
  • Work Hours/Day: 10
  • Exceptions: Public holidays, prayer days (Middle East)

Engineering 5-Day Office Calendar

Use Case: Design work, procurement, project management

  • Workweek: Monday-Friday working, Sat-Sun off
  • Hours: 08:00-17:00 with 1-hour lunch (8 working hours)
  • Work Hours/Day: 8
  • Exceptions: National holidays, year-end shutdown

Turnaround/Shutdown Calendar

Use Case: Refinery maintenance, plant turnarounds

  • Workweek: Monday-Sunday, all days working
  • Hours: 12-hour shifts (06:00-18:00 day shift, 18:00-06:00 night shift)
  • Work Hours/Day: 24 (two 12-hour shifts)
  • Duration: Temporary calendar for shutdown period only

Crew Rotation Calendars

Oil & gas offshore operations use rotation schedules. Model these in P6:

14/14 Rotation (Equal Time)

Crew works 14 days on, 14 days off

  • Create two resource calendars: Crew A and Crew B
  • Crew A: Days 1-14 working (24hrs), Days 15-28 non-working
  • Crew B: Days 1-14 non-working, Days 15-28 working (24hrs)
  • Pattern repeats every 28 days
  • Assign half the resources to each calendar for continuous coverage

28/28 Rotation

Crew works 28 days on, 28 days off (common for remote locations)

  • Create Crew A and Crew B calendars
  • 56-day repeating pattern
  • Ensures continuous operations with crew overlap during changeover

Assigning Calendars

Project-Level Assignment

  • Open project: File ‚Üí Open
  • Project ‚Üí Project Details ‚Üí Dates tab
  • Select Default Calendar from dropdown
  • All new activities inherit this calendar unless overridden

Activity-Level Override

  • Select activity in Activities view
  • In General tab or Activity Details window
  • Change Calendar field to different calendar
  • Example: Engineering activities use 5-day, construction uses 7-day

Resource-Level Calendar

  • Enterprise ‚Üí Resources
  • Select resource ‚Üí General tab
  • Assign calendar to resource
  • When resource assigned to activity, P6 considers resource calendar for availability

Calendar Hierarchy and Precedence

When multiple calendars apply, P6 follows this logic:

  1. Resource Calendar: If resource has calendar, it determines when resource can work
  2. Activity Calendar: Determines when activity can progress
  3. Project Calendar: Default if no activity calendar assigned
  4. Result: Activity can only progress when BOTH activity calendar AND all assigned resource calendars show working time
‚ö†Ô∏è Common Mistake: Assigning 24/7 calendar to offshore activities but forgetting to also assign 24/7 calendar to offshore resources. Result: Activities calculate as 8-hour days despite calendar showing 24 hours!

Weather Windows and Seasonal Calendars

For marine operations sensitive to weather (heavy lifts, subsea installation):

1Create Seasonal Calendar:

  • Base calendar: Offshore 24/7
  • Add exceptions for monsoon/winter months when operations impossible
  • Example: North Sea - November to March limited weather windows

2Apply to Weather-Sensitive Activities:

  • Assign seasonal calendar only to critical lifts, topsides installation, piling
  • Regular activities continue on standard 24/7 calendar

Calendar Maintenance Best Practices

üí° Annual Review: Update global calendars each year with:
  • New public holidays (governments announce annually)
  • Planned plant shutdown dates
  • Major religious observances (Islamic calendar shifts yearly)
  • Known project-specific constraints
üí° Naming Convention: Use descriptive, standardized names:
  • Good: "OFFSHORE-24x7-CONTINUOUS"
  • Good: "ONSHORE-6DAY-10HR"
  • Good: "ENGINEERING-5DAY-OFFICE"
  • Bad: "Calendar1", "NewCal", "Test"
‚ö†Ô∏è Critical: Never delete calendars still assigned to activities or resources. This corrupts project data. Instead, rename with "OBSOLETE-" prefix and create replacement calendar.

Problem Cause Solution
Activity duration calculating incorrectly Wrong calendar assigned Check activity calendar, verify work hours/day setting
Schedule spans weekends unexpectedly 7-day calendar when 5-day intended Verify project default calendar, update if needed
Resources showing overallocated on holidays Resource calendar missing exceptions Add holidays to resource calendar

Advanced: Multiple Shift Calendars

For projects with day/night shifts tracking separately:

  • Create "DAY-SHIFT-12HR" calendar: 06:00-18:00
  • Create "NIGHT-SHIFT-12HR" calendar: 18:00-06:00
  • Assign to activities based on shift performing work
  • Or assign to resources representing shift crews
  • Enables tracking productivity differences between shifts

Oil & Gas Industry Applications

Why P6 is Essential for Oil & Gas

The oil & gas sector requires precise scheduling due to high costs, regulatory compliance, harsh environments, and complex multi-phase operations. P6 provides the control needed for upstream, midstream, and downstream projects.

Typical Oil & Gas Project Types

  • Upstream: Exploration, drilling, offshore platform construction, subsea installations
  • Midstream: Pipeline construction, compression stations, storage facilities
  • Downstream: Refinery turnarounds, petrochemical plants, terminal expansions
  • LNG: Liquefaction plants, regasification terminals, FLNG vessels

Industry-Specific P6 Configuration

Calendar Setup for Offshore Operations

  • Create 24/7 continuous work calendar for offshore installations
  • Define rotation schedules (14/14, 28/28 patterns) for crew changes
  • Account for weather windows for critical lifts and hookup operations
  • Set up onshore 5/2 or 6/1 work weeks for engineering and procurement
  • Configure global exceptions for regulatory shutdowns and maintenance windows

WBS Structure for Offshore Platform Project

Typical hierarchy aligns with discipline and construction workface:

  • 1.0 Engineering (FEED, Detailed Design, Procurement Support)
  • 2.0 Procurement (Long-lead Equipment, Bulk Materials, Subsea Systems)
  • 3.0 Fabrication (Topsides Modules, Jacket Structure, Piles)
  • 4.0 Installation (Jacket Launch, Topsides Lift, Piling, Hookup)
  • 5.0 Commissioning (Pre-commissioning, MC, Performance Testing)
  • 6.0 Startup (First Oil, Handover, Demobilization)

Activity Coding for O&G Projects

Implement multi-dimensional codes for powerful reporting:

  • Discipline Code: CIVIL, STRUC, MECH, ELEC, INST, PIPE, HVAC
  • Area Code: Drilling Module, Wellhead Platform, Flare Tower, Quarters
  • Phase Code: Engineering, Procurement, Fabrication, Installation, Commissioning
  • Contract Package: EPC-01, FabYard-A, InstallVessel-B
  • Safety Critical: Hot Work, Confined Space, Lifting Operations
  • HSE Category: Permit to Work (PTW) types for regulatory tracking

Earned Value for Capital Projects

Configure P6 for rigorous EVM analysis required by operators and lenders:

  • Define Activity Types with Performance % Complete methods
  • Assign budget (BCWS) at activity or resource level matching cost accounts
  • Track actuals through timesheet integration or manual entry
  • Monitor SPI and CPI trends for early warning of overruns
  • Generate S-curves comparing planned vs. actual expenditure
  • Calculate EAC (Estimate at Completion) for forecast reporting to stakeholders

Turnaround & Shutdown Scheduling

Refinery turnarounds demand intense P6 capabilities due to tight timeframes and high daily costs:

  • Create detailed 24-hour shift schedules with handover protocols
  • Use minute-level durations for critical path analysis (e.g., 480 minutes vs 1 day)
  • Model permit-to-work dependencies as constraints between activities
  • Track scaffolding erection/dismantling as constraining resource
  • Implement daily look-ahead schedules with actual vs. plan variance tracking
  • Use what-if scenarios to model contingency plans for discovered defects
üí° Best Practice: For mega-projects ($1B+), maintain separate P6 databases for Engineering, Procurement, and Construction with cross-project links. This prevents performance degradation and allows discipline-specific scheduling depth.
